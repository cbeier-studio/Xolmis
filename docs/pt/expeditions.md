# Expedições

O módulo **Expedições** é usado para organizar e documentar saídas de campo realizadas para pesquisa ornitológica. Uma expedição representa um **período definido de trabalho de campo**, geralmente envolvendo uma equipe de pesquisadores, objetivos específicos e uma área geográfica. Ao registrar expedições, você pode agrupar levantamentos, capturas e observações em um contexto comum, facilitando o acompanhamento das atividades e resultados.

Abra o módulo Expedições no menu principal: **Trabalho de campo → Expedições**.

## Adicionando ou editando uma expedição

Ao criar ou editar um registro de expedição, os seguintes campos estão disponíveis:

| Campo | Obrigatório | Descrição |
| --- | --- | --- |
| **Nome** | Sim | Nome curto que descreve e identifica a expedição (ex.: "Pantanal 2025") |
| **Data de início** | Sim | Primeiro dia da expedição |
| **Data de término** | Sim | Último dia da expedição |
| **Projeto** |  | Projeto ao qual a expedição está vinculada (opcional) |
| **Descrição** |  | Breve descrição da expedição, incluindo objetivos, espécies-alvo ou área de estudo |

## Boas práticas

- **Use nomes claros**: escolha nomes de expedição curtos, mas descritivos (região + ano funciona bem).  
- **Relacione a projetos**: se a expedição fizer parte de um projeto maior de pesquisa, vincule-a para garantir rastreabilidade.  
- **Registre datas precisas**: datas de início e término são essenciais para organizar levantamentos e capturas cronologicamente.  
- **Adicione notas descritivas**: inclua objetivos, habitats-alvo ou detalhes logísticos no campo de descrição.  
- **Agrupe levantamentos relacionados**: todos os levantamentos realizados durante a expedição devem ser vinculados a ela para consistência.  

## Relação com outros módulos

As expedições estão interligadas a várias outras partes do Xolmis:

- **[Amostragens](surveys.md)**: cada levantamento pode ser vinculado a uma expedição, agrupando eventos de amostragem em uma mesma saída de campo.  
- **[Capturas](captures.md)**: aves capturadas durante uma expedição são automaticamente associadas aos seus levantamentos.  
- **[Projetos](projects.md)**: expedições podem ser vinculadas a projetos específicos, conectando o trabalho de campo a objetivos de pesquisa mais amplos.  
- **[Licenças](permits.md)**: expedições frequentemente exigem autorizações, que podem ser gerenciadas no módulo Autorizações.  
- **[Relatórios](print-data.md)**: dados de expedições podem ser resumidos em relatórios, úteis para agências de financiamento ou registros institucionais.  

Ao gerenciar expedições no Xolmis, você cria uma estrutura organizada para o trabalho de campo, garantindo que todos os levantamentos, capturas e observações estejam devidamente contextualizados dentro do esforço de pesquisa mais amplo.
